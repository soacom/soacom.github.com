APIVersion Object

Includes information about an API version.

This object is used by the following operations:

  • POST /api/apis/apis
  • PUT /api/apis/versions/{ApiVersionID}
  • GET /api/apis/versions/{APIVersionID}

The properties included in the APIVersion object are listed below.

Property Type Required Description
APIVersionID string Required The unique ID for the specified API version.
APIID string Required The unique ID for the specified API.
Name string Optional The name of the API.
Type string Optional

The type for the API version.

Valid values:

  • REST API: shttp
  • SOAP 1.1 API: soap11
  • SOAP 1.2 API: soap12
DefaultProfile string Optional

The default profile for the API version, if specified.

Only applicable for adding operations via the user interface.
Description string Optional The description of the API version.
Tag string[ ] Optional One or more user-defined tags assigned to the discussion.
ProductionEndpoint anyURI Optional The Production endpoint listed for the API version in the API Details page in the user interface.
ProductionEndpointOAuthDetails EndpointOAuthDetails Optional

The OAuth configuration details for the Production endpoint, such as OAuth version and URLs.

Used only in response messages.
SandboxEndpoint anyURI Optional The Sandbox endpoint listed for the API version in the API Details page in the user interface.
SandboxEndpointOAuthDetails EndpointOAuthDetails Optional

The OAuth configuration details for the Sandbox endpoint, such as OAuth version and URLs.

Used only in response messages.
Descriptor Descriptor Optional SOAP APIs only: Includes information about the WSDL that contains the technical details about the API version. For SOAP APIs, this is required in the request message when adding or updating an API version.
Operations Operations Optional Used for REST APIs only: Information about the operations that are valid for the API version.
Endpoints Endpoints Optional Information about the endpoints that are valid for the API version.
Policies Policies Optional

An array of information about one or more policies that are enforced for the API version, if applicable.

For a list of valid policies, see Client Authorization Policy Options.

To retrieve a list of available policies, including PolicyKey and PolicyType, use the GET /api/policies operation.
TargetAPI TargetAPI Optional Includes information about the target API—the destination endpoint, or next hop, defined when an API is using the platform as a proxy.
Visibility string Optional The visibility setting for the API; either Public, which is visible to all users, or Limited, which is visible only to API Administrators and Private API Group members.
Created dateTime Optional The date/time stamp when the resource was first created. Used only in response messages.
Updated dateTime Optional The date/time stamp when the resource was last updated. Used only in response messages.
ProductionEndpointAccessAutoApproved boolean Optional Indicates whether a request for production access for the API is automatically approved. Defaults to false; unless set to true, production access must be approved by an API Administrator.
SandboxEndpointAccessAutoApproved boolean Optional Indicates whether a request for sandbox access for the API is automatically approved. Defaults to true; unless set to false, sandbox access is automatic and immediate upon request.
RatingSummary RatingSummary Optional An aggregated, averaged summary of individual user ratings for the resource, including the current ratings count for each possible value (1, 2, 3, 4, or 5). Used only in response messages.
ProductionEndpointExists boolean Optional Indicates whether a Production endpoint exists for this API. For internal use only.
SandboxEndpointExists boolean Optional Indicates whether a Sandbox endpoint exists for this API. For internal use only.
