GET /api/apis/isendpointunique

Checks whether the specified endpoint is unique.

Authorization Roles/Permissions: To complete this operation successfully, a user must be logged in and must be an Administrator for the API.

HTTP Method

URL

https://<hostname>/api/apis/isendpointunique?url={url}&ApiVersionID={APIVersionID}

Sample Request

In the first sample URL below, the API is being added, so it doesn't yet have an APIVersionID. In the second example, the API is being edited, so the APIVersionID is included as a query parameter.

Request URL #1

https://<hostname>/api/apis/isendpointunique?url=http%3A%2F%2Fakmepayments_api.broker.soa.com

Request URL #2

https://<hostname>/api/apis/isendpointunique?url=http%3A%2F%2Fakmepayments_api.broker.soa.com&ApiVersionID=apiversion19123.open

Sample request headers

GET /api/apis/isendpointunique?url=http%3A%2F%2Fakmepayments_api.broker.soa.com%2Fa.com HTTP/1.1
Host: <hostname>
Accept: */*

Request Parameters

Parameter Parm Type Data Type Required Description
URL Query String Required The URL of the API version, for verification that it is unique.
APIVersionID Path string Required The unique ID for the specified API version.

Back to top

Response

If successful, this method returns HTTP status code 200, with a boolean value indicating whether the endpoint is unique.
